
## 产品背景
作为我的第一款 mac 软件，我希望vibe coding 一个小工具，它可以轻量化的运行在 mac 栏中，点击后可以以悬浮窗的形式展示，展示用户自己关心的几个软件的使用时长。
## 产品价值
我目前短暂的理解还是情绪价值吧，没有太大意义，就像屏幕使用时间一样，偶尔看一看。（是否刚安装之后弹个框提醒用户先去用，过段时间再来看之类的，好像之前看到过类似的交互）
比如figma 已经有Figma 活动日历（一款追踪你在 Figma 中花费时间的菜单栏应用：https://macmenubar.app/app/figma-activity-calendar），和 JJYing 的摸摸鱼计时器 （https://momoyu.app/），但是 figma 活动日历主要是记录你在 figma 中花费的时间，主要用的是github 的热力度计时器，时间维度是月，这个月每天在 figma 中提交了多少改动。而摸摸鱼计时器主要是记录每一天，今天你在不同软件上花费了多长时间。事实上摸摸鱼有了每天的数据，想出一个月度视图也很简单。但是他目前的核心似乎还是轻松态的表达每日“摸鱼时长”，这真的是一个很有吸引力的想法，也很有社区自发传播力。


那么我的机会点是什么呢？或者我需要的是一个什么样的软件呢？首先 figma 活动日历他只能追踪 figma，这一点很不好，现在 vibe coding 的时代，大家可能会在各个窗口：IDE、浏览器、figma、笔记软件等等直接来回切换。
那我做的是什么呢？是像摸摸鱼一样，让用户自己选择主力软件，然后记录他的多个主力软件每天使用时长，然后以活动日历中热力图的形式表达出来。
这个东西有价值吗？对我来说有价值吗？记录一下我今天在 figma 中花了多久，在 cursor 中花了多久，意义是什么呢？那摸摸鱼的意义是什么呢？摸摸鱼的意义更直观，用户很轻松就可以看到：“诶，今天是有认真工作的！”。
那 github 中的热力图意义是什么呢？意义是让用户看到自己成长的轨迹？还是工作的勤奋程度？
